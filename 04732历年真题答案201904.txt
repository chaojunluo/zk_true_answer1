在中断过程中，中断识别号通常通过数据总线送入 CPU。这是因为数据总线用于在系统的各个部分之间传输数据，包括 CPU、内存和 I/O 设备。当 CPU 响应某一中断源的中断请求后，相应的中断识别号就会被放置在数据总线上，然后由 CPU 读取。这样，CPU 就可以知道是哪个设备发出了中断请求，并采取相应的行动
。

控制总线主要用于在 CPU 和其他硬件设备之间传输控制信号，这些控制信号包括读取、写入、中断等指令。而数据总线则用于传输实际的数据，包括从 CPU 到内存的数据，从内存到 CPU 的数据，以及中断向量或中断识别号等。

当 CPU 响应一个中断请求时，它需要知道是哪个设备发出的请求，这就需要通过中断识别号来实现。这个识别号是一种数据，所以它通过数据总线传输到 CPU。如果通过控制总线传输，那么 CPU 就无法区分这是一个控制信号还是一个数据信号。

因此，虽然控制总线在中断过程中起着重要的作用（例如，发送一个信号来通知 CPU 有一个中断请求），但是中断识别号还是通过数据总线送入 CPU 的。这样可以确保 CPU 正确地识别和处理中断请求。

栈顶内容由硬件自动返回到IP、CS和FLAGS，是由于CPU执行了IRET指令。IRET（中断返回）指令用于从中断处理程序返回到被中断的程序。
当执行IRET指令时，CPU会从栈中弹出保存的FLAGS、CS和IP寄存器的值，并将它们恢复到相应的寄存器中，以便继续执行被中断的程序。这个过程是由硬件自动完成的，无需程序员手动干预。
